# Tool library
 
This is the canonical reference for every tool the skill suggests. Each entry has:
 
- **Category:** What the tool does
- **Typical stages:** Which framework steps it fits
- **Brand color:** Hex color for the dot/tile (used in the diagram and artifact)
- **Brand letter:** 1-3 letters for the tile icon when no SVG logo is available
- **Typical learning time:** Used inline in the onboarding next to "willing to learn" so users have context to decide. E.g., "weekend project", "1 week", "2-3 weeks".
- **Swap for:** Priority-ordered alternatives if user picks "not interested"
- **Learning path:** 1-3 short, named resources for users marking "willing to learn"
When in doubt, pick the swap from the same category, then drop a tier in capability before swapping categories entirely.
 
## Specialist tools (prefer over generalists where they fit)
 
These are specialists the diversity rule should reach for before defaulting to a generalist like Claude or Notion.
 
### Perplexity
- **Category:** AI-powered research and competitive scan
- **Brand color:** `#1F1F1F` (dark)
- **Brand letter:** Px
- **Typical stages:** Discovery, Strategy/positioning, ICP and competitor research
- **Why over Claude:** Perplexity grounds responses in cited web sources — ideal for "scan the gardener app market" or "what do users say about competing tools" work where sourced info matters.
- **Swap for:** Claude (with web search), ChatGPT (with browsing)
- **Learning path:**
  - Perplexity tutorials — https://www.perplexity.ai/help
### Anthropic Console
- **Category:** AI prompt-testing playground (developer-facing)
- **Brand color:** `#DA7756`
- **Brand letter:** AC
- **Typical stages:** AI behavior testing, AI confidence + error states
- **Why over Claude (chat):** Console has system-prompt fields, model-version selectors, side-by-side comparison, and saved prompt versions. Right tool for "test 50 prompt variants and compare outputs" work, where chat is the wrong shape.
- **Swap for:** OpenAI Playground, Vercel AI Playground
- **Learning path:**
  - Anthropic Console docs — https://docs.anthropic.com/en/docs/build-with-claude/prompt-engineering/overview
### Anima
- **Category:** Figma-to-code specialist (with React, Vue, HTML output)
- **Brand color:** `#A259FF`
- **Brand letter:** An
- **Typical stages:** Code-ready handoff
- **Why over Cursor or Figma alone:** Anima's specifically built for the design-to-code translation step — auto-generates production-quality components from Figma frames. Pair with Cursor for AI-assisted refinement.
- **Swap for:** Builder.io, Locofy, Figma Dev Mode
- **Learning path:**
  - Anima docs — https://docs.animaapp.com
### Linear
- **Category:** Project + issue tracking (engineering-aware)
- **Brand color:** `#5E6AD2`
- **Brand letter:** L
- **Typical stages:** Code-ready handoff, Build/iterate, Launch + iteration
- **Why over Notion:** Linear is purpose-built for sprint cycles, issue tracking, and the rhythm of post-launch iteration. Notion is great for docs, weak for tracking.
- **Swap for:** Asana, Jira, Shortcut
- **Learning path:**
  - Linear method (their philosophy) — https://linear.app/method
  - Linear docs — https://linear.app/docs
### Mixpanel
- **Category:** Product analytics + funnel analysis
- **Brand color:** `#7856FF`
- **Brand letter:** Mp
- **Typical stages:** Launch + iteration
- **Why alongside or instead of Posthog:** Mixpanel is the classic specialist for funnel + retention analysis. Use as a swap candidate when Posthog is already familiar, or as a comparison point.
- **Swap for:** Amplitude, Posthog, Heap
- **Learning path:**
  - Mixpanel Bootcamp (free) — https://mixpanel.com/blog/mixpanel-bootcamp/
