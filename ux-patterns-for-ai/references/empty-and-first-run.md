# Empty and first-run

Before the user has used the feature, they don't know what to ask. The empty state is where adoption lives or dies — most AI features lose more users at first-touch than at any other moment.

**Three things to show in an empty state:**

1. **A one-line value proposition.** Not "Ask anything" — something specific. "Ask anything about your meeting notes" or "Generate a cold email from a job description." Specific framings teach the user what the product is for.

2. **Three suggested prompts that demonstrate range.** Pick prompts that (a) actually work well, (b) showcase different capabilities, and (c) produce visibly impressive results on the first try. The empty state is your demo; act like it.

3. **One piece of context or social proof.** "Used 12 times this week by your team." "Last query: 'summarize Q3 retro.'" Social proof and recency make the AI feel embedded in the user's actual work, not abstract.

**The first run produces a calibration moment.** If the first answer is mediocre, the user concludes the product is mediocre — even if subsequent answers would be great. Treat the first run as a marketing surface: the prompts, the response time, the formatting, the citations all need to be right.

**Anti-patterns to flag:**
- Empty illustrations with no actionable element. Decorative wallpaper. The user is here to use the feature, not to admire your art.
- Generic suggested prompts ("Tell me a joke," "Help me brainstorm") in a product-specific tool. Sets the wrong expectations and undersells what the product actually does.
- Long onboarding tour before the user can try the feature. By the third tooltip, they've left.
- Suggested prompts that don't actually work well in practice. Test them — first impressions calibrate trust permanently.
- Empty state copy that explains how the AI works ("powered by GPT-4..."). Users don't care; they want to see what it does for them.

**The decision the designer needs to make:** What's the one specific thing the user should try first, and does it produce a result that earns their continued attention?

**Revisit if:** First-session abandonment is high (the empty state isn't getting them past the blank-page problem) or the second-session retention is low (the first run didn't earn their trust — the suggested prompts are probably underdelivering).
